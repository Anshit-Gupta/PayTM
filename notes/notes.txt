# const session = await getServerSession(authOptions); why we pass the authoptions here ??
We pass authOptions to getServerSession so it knows how to configure authentication for your app. The authOptions object contains settings like providers, callbacks, and session handling rules. By passing it, getServerSession knows how to validate the user's session, which authentication providers to use, and how to fetch user data securely. This ensures the session is created and validated according to your app's specific authentication setup.

# session?.user?.id; why does we add question marks here
The question marks (?.) are called optional chaining. They safely access properties that might not exist.
